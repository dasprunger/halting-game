Tic-Tac-Toe

Phase 1

Make graphics methods and test them using the developer tools. For this project, I wrote methods to draw an X, draw an O, and clear a square, then tested them out.

Phase 2

Decide how you're going to represent your game. You need some variables so that the computer can keep track of what happens behind the scenes. For this, I decided I would keep an array of 9 items (all of which are "b" for blank, "x" for X, and "o" for O) to represent the board. (Check out the "currentPosition" variable in the Javascript.)

Phase 3

Write some functions to deal with the game for you. For this, I needed to write a function to see if it was legal to put a mark in a square, to actually put the mark in the square, to figure out whose turn it was, and to check if someone had won.

Phase 4

Wire up the game logic and the display. Usually for this you'll need to write some mouse click handlers or maybe use the prompt() function. If you're here, talk to me!
